## 📖 项目介绍

### 这个项目是什么

**咖啡商城小程序**是一个面向咖啡店的**扫码点单系统**，包含以下核心功能：

- 🛒 **扫码点单** - 顾客扫桌面二维码进入小程序，浏览商品、加入购物车、在线支付、实时履约（取餐号、预计等待时间、催单）
- 👨‍💼 **商家后台** - 实时接收订单、管理商品、订单履约管理（接单、制作、叫号）
- 💳 **会员钱包** - 余额充值、消费流水、充值优惠模板、会员等级折扣
- 🎯 **营销活动** - 满减活动、指定商品优惠、活动时段控制
- 🎉 **线下活动** - 活动发布、用户预约报名、状态管理
- 🎨 **AI图片润色** - 商家上传商品图片，AI自动优化图片质量
- 💰 **多种支付** - 支持余额支付、微信支付（收钱吧）

### 技术栈

**后端**
- Spring Boot 2.x
- MyBatis
- MySQL 8.0
- Redis
- Shiro（权限管理）
- Quartz（定时任务）

**前端**
- uni-app（小程序）
- Vue.js
- Thymeleaf（后台管理页面）

**第三方服务**
- 微信小程序（登录、支付、订阅消息）
- 腾讯云COS（对象存储）
- 收钱吧支付
- AI图片处理服务

## 🚀 项目怎么使用

### 环境要求

- JDK 1.8+
- MySQL 8.0+
- Redis 3.0+
- Maven 3.6+
- Node.js 14+
- 微信开发者工具

### 后端启动步骤

#### 1. 克隆项目
```bash
git clone https://github.com/你的用户名/coffee-shop.git
cd coffee-shop
```

#### 2. 创建数据库

详细的数据库初始化说明请查看：[sql/README.md](sql/README.md)

```bash
# 登录MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE ruoyi DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE ruoyi;

# 执行若依框架SQL
mysql -u root -p ruoyi < sql/ry_20210924.sql

# 执行定时任务框架SQL
mysql -u root -p ruoyi < sql/quartz.sql

# 执行业务表SQL（合并版，一次搞定）
mysql -u root -p ruoyi < sql/coffee_all_business.sql
```

#### 3. 修改配置文件

编辑 `src/main/resources/application.yml` 和 `application-druid.yml`

```yaml
# 数据库配置
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/ruoyi?useUnicode=true&characterEncoding=utf8
    username: root
    password: 你的数据库密码
    
# Redis配置
redis:
  host: localhost
  port: 6379

# 腾讯云COS配置（建议使用环境变量）
cos:
  secret-id: ${COS_SECRET_ID}
  secret-key: ${COS_SECRET_KEY}
```

#### 4. 启动项目
```bash
# 使用Maven启动
mvn clean install
mvn spring-boot:run

# 或在IDE中运行 RuoYiApplication.java
```

#### 5. 访问后台
```
URL: http://localhost:8080
账号: admin
密码: admin123
```

### 小程序端启动步骤

#### 1. 进入小程序目录
```bash
cd RuoYi-AbuCoder-UniAppWx/Ruoyi-AbuCoder-UniApp-WX
```

#### 2. 修改API地址
编辑 `utils/apiconfig.js`
```javascript
export default {
  baseUrl: 'http://localhost:8080'  // 改为你的后端地址
}
```

#### 3. 导入微信开发者工具
- 打开微信开发者工具
- 导入项目目录
- 填写AppID
- 编译运行

## ⚠️ 已知问题与风险

### 🔴 高优先级问题（P0）

#### 1. 支付回调可能丢失
- **发生概率**：约2-3%
- **影响**：用户付款但订单未更新
- **当前状态**：部分解决（前端轮询）
- **待完善**：后端主动查询 + 定时任务补偿

#### 2. 密钥安全存储
- **现状**：配置文件直接写密钥
- **风险**：代码泄露导致密钥泄露
- **改进方案**：使用环境变量或密钥管理服务

#### 3. 订阅消息授权机制
- **机制**：微信小程序订阅消息为一次性授权
- **说明**：用户授权一次只能发送1条消息，消息发送后授权失效
- **影响**：用户每次下单都需要重新授权才能收到取餐通知
- **处理**：已优化为支付成功页统一请求授权，避免重复弹窗

### 🟡 已解决问题 ✅

#### 1. 订单金额被篡改 ✅
- **解决方案**：后端重新计算所有金额，忽略前端传入的金额参数
- **实现位置**：`OrderApiController.java` 和 `ScanOrderServiceImpl.java`

#### 2. 并发超卖 ✅
- **解决方案**：SQL WHERE条件：`UPDATE ... SET stock = stock - #{quantity} WHERE stock >= #{quantity}`
- **实现位置**：`TProductMapper.xml`

#### 3. 会员折扣动态配置 ✅
- **解决方案**：从 `sys_config` 读取折扣率，数据库字段仅作快照
- **实现位置**：`MemberService.java`

**详细问题清单**见：[问题梳理与风险清单](docs/问题梳理与风险清单.md)
